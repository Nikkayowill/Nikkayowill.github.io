Boniface & Paulina Okafor Memorial Hospital

An ASP.NET Core MVC hospital management platform connecting public hospital information, patient services, clinical workflows, and administration in one application.

Role
Full-stack developer
Status
In development
Year
2026
Platform
.NET 10 / SQL Server

01 / Problem

One platform for connected hospital services.

The application addresses fragmented communication and manual workflows by giving patients, clinicians, and administrators structured tools for accessing information, requesting care, and managing daily hospital operations.

02 / Features

Public access, patient care, and administration.

Public pages introduce the hospital’s services, doctors, and departments. Authenticated experiences support appointment requests, teleconsultations, patient notifications, payments, donations, and operational management.

  • Public hospital pages
  • Appointment requests
  • Teleconsultations
  • Secure patient portal
  • Administrative dashboard
  • Doctor and department management
  • Payments and donations
  • Real-time notifications

03 / Architecture

Server-rendered foundations with focused interactivity.

The platform uses ASP.NET Core MVC and Razor Views on .NET 10, with Entity Framework Core and SQL Server for relational data. ASP.NET Core Identity provides authentication and role-based authorization across patient, doctor, and administrator workflows.

Tailwind CSS and Bootstrap 5 support the responsive interface, Alpine.js handles lightweight client-side interactions, and SignalR enables real-time consultation and notification updates. Progressive Web App support improves installability and resilience when connectivity is limited.

04 / Engineering

Clear permissions and resilient interactions.

One challenge was enforcing permissions across multiple user roles without spreading authorization rules throughout the application. Centralized policies and separated service boundaries keep access decisions consistent and business logic maintainable.

Another challenge was preserving a useful experience during inconsistent connectivity. Progressive caching, explicit connection-state handling, and synchronization-aware interfaces provide limited offline support without hiding the state of patient data from users.
