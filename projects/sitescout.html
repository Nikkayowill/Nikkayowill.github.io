SiteScout

A sales pipeline app for a local web developer turning small-business prospects into tracked leads, timely follow-ups, tailored pitches, and priced website opportunities.

Role
Full-stack developer
Status
Product build
Year
2026
Platform
Next.js / Supabase
Next.js App Router TypeScript Tailwind CSS Supabase Server actions
Private product build

01 / Problem

Turning scattered outreach into a repeatable sales system.

Local website sales can quickly spread across notes, browser tabs, screenshots, drafts, and memory. SiteScout brings that process into one focused workspace so each prospect has a clear status, next action, pricing context, and conversation history.

02 / What I did

Built the sales workflow from lead capture to pitch prep.

  • Designed dashboard, lead management, follow-up, pitch, and settings flows
  • Implemented server actions with service-layer Supabase database access
  • Created pricing defaults and reusable outreach template settings
  • Structured the app so branding and product direction can evolve cleanly

03 / Product

Leads, follow-ups, pitches, and pricing in one place.

The app supports the full early sales loop: finding leads, saving business details, tracking interactions, preparing outreach copy, managing follow-ups, and moving deals through a simple pipeline.

  • Dashboard metrics
  • Lead management
  • Lead finder workflow
  • Lead detail pages
  • Follow-up queue
  • Pitch generator
  • Interaction history
  • Pricing and template settings

04 / Engineering

Server actions with a clean database boundary.

SiteScout is built with the Next.js App Router, TypeScript, Tailwind CSS, Supabase, and server actions. Database operations are routed through a service layer so the interface, actions, and persistence logic stay easier to reason about as the product grows.

The app keeps branding intentionally neutral, making room for the product identity to evolve without tying the technical foundation to temporary naming.

05 / Outcome

A practical tool for selling with more consistency.

The default pricing model starts with a $500 one-time build and $20/month support plan, giving the app a real sales context while keeping the settings flexible for future packages, templates, and pitch styles.
